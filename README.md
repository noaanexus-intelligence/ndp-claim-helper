# ตัวช่วยเบิก NHSO Digital Platform (NDP) สำหรับ รพ.สต.

เครื่องมือช่วยจำและตรวจสอบเบื้องต้น สำหรับการขอรับค่าใช้จ่ายบริการสาธารณสุขผ่าน **NHSO Digital Platform (NDP)** — ลดปัญหาเคลม "ติด C" / ถูกตีกลับ ตั้งแต่ต้นทาง

**เปิดใช้งาน:** เปิดไฟล์ [`index.html`](./index.html) ในเบราว์เซอร์ได้เลย (หรือ deploy เป็นเว็บ — ดูด้านล่าง)

---

## ทำอะไรได้บ้าง

เครื่องมือเป็นหน้าเว็บเดียว (single-file) แบ่งเป็น 4 ส่วน:

| ส่วน | ใช้ทำอะไร |
|------|-----------|
| **รหัสเบิก PP** | ค้นหากิจกรรม PP Fee Schedule แล้วดูว่าต้องจับคู่ **ICD-10 + รหัสตรวจ (TMLT / CSMBS / NHSO / TMT)** อย่างไรให้ 1 ชุดเบิกสมบูรณ์ พร้อมตัวอย่างและปุ่มคัดลอกชุดรหัส |
| **ถอดรหัส Error** | พิมพ์ข้อความ error ที่ระบบแจ้ง (เช่น `ProviderType`, `Auth Code`, `INVOICE_NO`, `Token`) แล้วดูสาเหตุ + วิธีแก้ + เมนูที่ต้องไปตั้งค่า |
| **Checklist ตั้งค่า** | เช็กลิสต์ตั้งค่า HOSxP XE PCU ก่อนส่ง 13 แฟ้ม Plus (จำสถานะติ๊กไว้ในเครื่อง) |
| **อ้างอิง CODESYS** | ตารางระบบรหัส 001-TMT / 002-TMLT / 004-CSMBS / 005-NHSO ฯลฯ และหมวด BILLGRCS |

### ความเป็นส่วนตัว (PDPA)
ทุกอย่างทำงานใน **เบราว์เซอร์ของผู้ใช้ทั้งหมด** — ไม่มีการส่งข้อมูลผู้ป่วยออกนอกอุปกรณ์ ไม่มี backend ไม่มีการเก็บ log บนเซิร์ฟเวอร์

---

## สถานะปัจจุบัน (MVP)

- ครอบคลุมกิจกรรม PP Fee Schedule **~40 กิจกรรม** (จากทั้งหมด 105) โดยเลือกกลุ่มที่ใช้บ่อยก่อน:
  ฝากครรภ์/หลังคลอด, คุมกำเนิด, คัดกรองมะเร็ง (ปากมดลูก/ลำไส้/เต้านม), วัณโรค, เบาหวาน/ไขมัน/โลหิตจาง, ตับอักเสบ, วัคซีน
- ตัวถอดรหัส Error ครอบคลุม error หลักที่พบในคู่มือ/หน้างาน

### แผนพัฒนาต่อ
- [ ] เติมกิจกรรม PP ให้ครบ 105 (ข้อ 80–105 และรายละเอียดรหัสยา TMT แบบเต็ม)
- [ ] เพิ่มโหมดตรวจไฟล์ 43 แฟ้มที่ export มา (ตรวจ error ก่อนส่งจริง)
- [ ] เพิ่มโมดูลนวดแผนไทย / สิทธิ OFC-LGO

## การนำไปใช้งาน (Deploy)

เป็นไฟล์ static ไฟล์เดียว ไม่ต้องมี build step:

- **เปิดในเครื่อง:** ดับเบิลคลิก `index.html`
- **GitHub Pages:** Settings → Pages → Deploy from branch `main` → root
- **Vercel / Netlify:** Import repo นี้ แล้ว deploy ได้เลย (ไม่ต้องตั้งค่า build)
